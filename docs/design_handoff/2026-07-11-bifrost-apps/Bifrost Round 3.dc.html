1a

Design system for the app family

grounded in Theme.swift — keep · extend · replace
KEEP
Spacing scale — xs 4 · sm 8 · md 16 · lg 24 · xl 32. Sound; dense by default.
Radii — card 14 / control 10. Native iOS geometry; do not import the web kit's sharper radii.
Dynamic Type — all styles stay system text styles. The theme colors below are asset-catalog colors, so dark mode holds by construction.
SF Symbols — kept. No custom icon set.
EXTEND
Type roles — add New York (serif) for note titles and the wordmark only; add monospaced Dynamic Type for paths, uuids, timestamps — this user lives in markdown, metadata is first-class.
Component set — B1 has 4 components. Add the 8 below; every lens composes from these, nothing hand-rolled.
Lens scaffold — one shared load/empty/error/save container (right).
REPLACE
Color tokens — `accentColor`/orange/green become a named semantic palette (below): gold = principal's affordances, agent blue = machine-authored, vault green = record/sync, amber = staged, cyan = the two granted live bends. Never decorative.
Bare red error Text — replaced everywhere by the banner pattern with retry (O3).
Typed-id inputs — removed from the product (O1). Steering is a gesture on a visible item.

1b

As-built B1 — two-axis journey grades

A = workflow intuitiveness · B = implementation quality
S1 EnterA: B+ · B: B
Auth gate and visual vault picker are the strongest B1 screens — the dyslexia-safe pattern held. But landing dumps you into a raw list: no "where am I, what's new" orientation, no last-refresh anchor.
S2 Review attentionA: D · B: C
Typing an item id into a text field (O1) violates the ruled no-typed-identifiers constraint outright, and the audit trail can misstate the human's act (D1) — a trust defect in the one journey that exists to build trust. Highest-leverage fix in the app.
S3 Steer interestsA: C · B: C+
Raw 0–1 sliders expose the model's internals without its reasoning — no "because you did X" evidence, so steering is blind. The Never list is a dead end (O6).
S4 Confirm entityA: C− · B: B−
Merge silently takes the first candidate (O7) and nothing offers undo (O5) — both collide with reversible autonomy. Candidate IDs are shown as raw strings the user must mentally resolve.
S5 Read & editA: B · B: C+
Browse→render→edit→save works, but the editor has no dirty state, no cancel, no save confirmation — and D3 (leading-zero YAML mutation) is silent data change, against ruling 8. Conflict copies exist on disk with no surfacing.
S6 Consent glanceA: B+ · B: B
Deliberately light and honestly read-only — correct under posture A. Needs only the shared scaffold, provenance footer, and a clear statement of what "dormant in v1" means.
UPLIFT MOVES, RANKED BY LEVERAGE
  1. Steering as gestures on visible items (S2) — swipe attended/skipped on the item itself; truthful audit line per act. Removes O1, closes D1's surface.
  2. The shared lens scaffold — one load/empty/error/save container adopted by all six lenses. Removes O2, O3, O4 in a single component.
  3. Undo bar on every consequential act — merge/reject, mark attended/skipped, save. Makes ruling 7 visible. Removes O5.
  4. Explicit candidate choice on phone merge (S4) — merge disabled until a candidate is picked. Removes O7 (advisory: changes shipped B1 behaviour, see 1f).
  5. Editor honesty (S5) — dirty dot, cancel/discard, saving→saved progression, save-failure keeps text with Retry/Copy, conflict-copy banner; D3 surfaced as a pre-save warning, never silent.
  6. Identity layer — app icon, launch screen, the Yggdrasil dark theme verified (O8, O9).

1f

"Markdown holds the record" — per-surface stance · advisory flags

Mimer iPhone
Every lens ends in a YggSourceFooter naming its note, with "Open in Obsidian". The two bends carry the cyan LIVE badge instead — the absence of a backing note is stated. Editing shows raw markdown; the app renders, it never owns.
Mimer iPad
The inspector is the record made visible — frontmatter verbatim, absence stated. Annotations and promotions preview the exact markdown block they will append before committing. Entity decisions are shown as appended log lines, undo included.
Heimdal
Authors only its device note, named on the Device screen. The queue is the folder — relaunch rebuilds it from disk. "Delivered" is defined on screen as "placed in the folder". Telemetry is the granted bend, badged LIVE, persisting nothing.
ADVISORY — OWNER RULING NEEDED (NO SILENT REDESIGNS)
  1. Tab topology: shown as 5 tabs (Today · Interests · Entities · Vault · Heimdal) with Consent + Settings behind the gear. The B1 shell specs 6 tabs; adding Heimdal as a 7th degrades all of them. Conflicts with the as-built tab list.
  2. Phone merge requires explicit candidate choice (S4): the iPad spec removes the first-candidate default; I propose the phone inherits that now. Changes shipped B1 behaviour.
  3. Interests Never list gains Restore (S3): B1 specs it read-only. Restore is one-gesture reversal of a prior steering act, consistent with ruling 7 — but it is an AC change.
  4. D3 pre-save warning (S5): "Keep as text / Save anyway" adds a decision point the editor spec doesn't have; alternative is fixing YAML round-tripping hub-side and no UI at all.
Everything else stays inside the walls: posture A operative, ASR hub-side, no transcript on device, no typed paths, no push, failure always visible.
